Deployment checklist (Madni Care app — static front end + optional API)

1) Build locally
   npm ci
   npm run build
   Output: dist/

2) Environment variables (set in Netlify / Vercel / your host UI, or .env.production)
   VITE_BASE           — base path if not site root (e.g. /repo-name/ on GitHub Pages). Empty for root.
   VITE_API_BASE_URL   — full API origin in production, no trailing slash (e.g. https://api.example.com).
                         Leave empty in dev; Vite proxies /api to the local backend.

3) SPA routing
   Netlify: netlify.toml rewrites are configured.
   Vercel: vercel.json rewrites are configured.
   Apache: public/.htaccess is included when you copy dist/ + public rules to the server.

4) Backend (server.ps1)
   Static hosts only serve the React build. Run the PowerShell API on a separate machine or service,
   then set VITE_API_BASE_URL to that origin. CORS must allow your front-end origin.

5) Smoke test after deploy
   Open site, catalog loads; admin/API features need step 4 if you use the JSON API.
